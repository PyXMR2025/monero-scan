# deployment/README: Dockerise
author: plowsof
created_at: 2025-03-22T04:02:41Z
status: open

to test:
`git fetch origin`
`git pull origin head pull/2454/head`

1. Navigate to your local `monero-site` git repository.
2. build the Docker container with `docker build --no-cache -t getmonero-container .`
3. to generate sile files to `_site` run `docker run -v $(pwd):/srv/jekyll getmonero-container bundle exec jekyll build` (Output dir can be changed by adding `--destination <output dir>`
    - for getmonero maintainer : confirm file permissions of the `_site` folder do not interfere with deployment, you can pass `--destination _test` if you want to confirm.
5. Then serve with: `docker run -p 4000:4000 -v $(pwd):/srv/jekyll getmonero-container`. If you want, you can speedup this process by loading only the last blog post instead of all of them. Add `bundle exec jekyll serve --limit_posts 1 --host 0.0.0.0` to the command above.
6. Open a browser and go to [http://127.0.0.1:4000](http://127.0.0.1:4000).
7. If all went well, you should see the Monero website and you're ready to make changes.

After running the cmd below, the Gemfile.lock in your local git should remain unchanged (unless there are updates available)
1. `docker run -v $(pwd):/srv/jekyll getmonero-container bundle update`
