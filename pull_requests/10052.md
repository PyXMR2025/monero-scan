# monero_blockchain_export: >2.5x blocksdat export speed
author: jeffro256
created_at: 2025-08-21T06:10:20Z
status: open

Uses efficient DB traversal to slash disk read time, which is a large chunk of the run-time.

Went from 2.175s to 0.849s with an internal NVME SSD and an Intel i7-1355U CPU (both had buffer cache populated before each run). Results may vary (hopefully positively) when using hard disks, external SSDs, etc.

To test, run `monero-blockchain-export --blocksdat --output-file export_blocksdat_test.dat --block-stop X` where `X` is a high block height that exists in your DB. Run it with both the old and new binary to compare performance. You can even put them in separate output files and run `sha256sum` on both of them to check equality.
