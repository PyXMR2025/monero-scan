# cmake: don't redefine static_assert for cxx
author: tobtoht
created_at: 2025-11-24T21:03:01Z
status: merged

Fixes #10230.

Building with `-DSANITIZE=ON` sets `-fsanitize=address,undefined`, which causes the `try_compile` for `cmake/test-static-assert.cpp` to fail if `-lasan` is missing.

This causes `-Dstatic_assert=_Static_assert` to be added to `CXX_FLAGS` (see diff), which doesn't make sense given that this is a [C11 keyword](https://en.cppreference.com/w/c/language/_Static_assert.html) that doesn't exist in C++.
