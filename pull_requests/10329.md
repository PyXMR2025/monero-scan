# tests: add mempool throughput benchmark
author: DUQUEredes
created_at: 2026-02-21T22:12:58Z
status: closed

## Summary

Add `speed_mempool.py` to functional tests, a benchmark that measures mempool performance in regtest mode:

- **TX creation time** — `wallet.transfer` with `do_not_relay` to isolate wallet-side construction
- **Mempool insertion throughput** — `send_raw_transaction` with 20 pre-built transactions, measuring ~95 txs/sec on commodity hardware
- **Mempool query latency under load** — `get_transaction_pool`, `get_transaction_pool_hashes`, `get_transaction_pool_stats` at pool sizes 0, 10, and 20
- **Block template fill time** — `generateblocks(1)` with a loaded mempool

Follows the pattern established by `speed.py`. No time assertions (hardware-dependent), only correctness assertions.

### Design notes

- Mines 130 blocks to ensure sufficient unlocked coinbase outputs across all benchmark stages
- Uses relay-then-flush-then-resubmit for the mempool insertion benchmark (isolates daemon insertion time from wallet construction)
- Resets wallet state between stages to avoid key image conflicts from unrelayed transactions
