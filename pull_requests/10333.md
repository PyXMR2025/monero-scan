# tests: add mempool throughput benchmark
author: DUQUEredes
created_at: 2026-02-23T11:09:12Z
status: closed

## Summary

- Add `tests/functional_tests/speed_mempool.py`, a benchmark for mempool operations in regtest mode
- Measures TX creation time, mempool insertion throughput, query latency under load, and block template fill time
- Complements the existing `speed.py` benchmark with mempool-specific metrics

## What it measures

| Benchmark | What it times | Why it matters |
|-----------|--------------|----------------|
| TX creation | `wallet.transfer()` with `do_not_relay` | Wallet-side overhead per transaction |
| Mempool add | `send_raw_transaction()` | Core insertion path: validation, key image tracking, sorting |
| Mempool queries | `get_transaction_pool`, `pool_hashes`, `pool_stats` at 0/10/20 txs | RPC latency scaling with pool size |
| Block template fill | `generateblocks(1)` with loaded mempool | TX selection + block construction time |

## Design notes

- Uses relay-then-flush-then-resubmit pattern to isolate mempool insertion time from wallet TX construction
- Resets wallet state between stages to avoid key image conflicts
- No time assertions — only correctness assertions (pool sizes, TX hashes). Timing is reported, not enforced
- Follows existing `speed.py` conventions (standalone benchmark, not in `functional_tests_rpc.py` default suite)
