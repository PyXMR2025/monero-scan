# fix: return early in sensitive methods for --restricted-rpc
author: SNeedlewoods
created_at: 2026-03-23T16:31:12Z
status: merged

Some `monero-wallet-rpc` methods should not be allowed with `--restricted-rpc`.
These methods were identified:
`on_set_subaddr_lookahead`
`on_create_address`
`on_label_address`
`on_create_account`
`on_label_account`
`on_tag_accounts`
`on_untag_accounts`
`on_set_account_tag_description`
`on_get_tx_key`¹
`on_get_tx_proof`¹
`on_get_spend_proof`¹
`on_get_reserve_proof`¹
`on_export_key_images`
`on_start_mining`
`on_stop_mining`
`on_create_wallet`
`on_open_wallet`
`on_close_wallet`

¹: Didn't restrict the proofs, but will do if someone provides good arguments, that I didn't see. (Edit:  also didn't restrict `get_tx_key`)

I also added the following "state-changing" commands to that list:
`on_freeze`
`on_thaw`
`on_generate_from_keys`
`on_restore_deterministic_wallet`

I'm unsure if `relay_tx` should be allowed, feedback is welcome.

Wouldn't change these in this PR, just wondering why the following commands are restricted:
`on_describe_transfer`
`on_get_attribute`
`on_get_transfers` ²
`on_get_transfer_by_txid` ²
(² Edit: see #10271)

Should `on_query_keys` be allowed with `--restricted-rpc` for `view-only` wallets? Currently it isn't [src](https://github.com/monero-project/monero/blob/b9998fc9e1c280d30e4b40ba983b1964eb256d66/src/wallet/wallet_rpc_server.cpp#L2267-L2272).
