## Summary

Closes the feature request discussed in #2274. Addresses the bounty at https://bounties.monero.social/posts/32.

## Approach

Rather than embedding libi2pd as a source submodule (as in #4263 and #4337), this PR bundles pre-built i2pd binaries and manages the process via \`QProcess\` — the same pattern used for P2Pool. This keeps build complexity minimal, avoids inflating the GUI binary, and lets i2pd be updated independently.

## What's changed

- \`src/i2p/I2pManager.cpp/h\` — manages i2pd lifecycle (start/stop/restart), cross-platform binary detection, real \`isRunning()\` state via QProcess signals
- \`cmake/I2pBundler.cmake\` — arch-aware binary selection at build time (macOS, Linux x86_64, Linux aarch64, Windows), SHA256 hash verification before copy
- \`external/i2p/bin/\` — pre-built i2pd binaries for all four platforms
- \`pages/settings/SettingsNode.qml\` — enable toggle, start/stop button, live status display, data dir picker, HTTP/SOCKS/SAM port config, autostart toggle, anonymous-inbound address field
- \`main.qml\` — \`appendI2pDaemonArguments()\` injects \`--tx-proxy i2p,127.0.0.1:7656\` and optionally \`--anonymous-inbound\` into monerod args automatically when i2p is enabled
- \`installers/windows/Monero.iss\` — i2pd.exe included in Windows installer

## User experience

Settings → Node → check "Enable I2P router" → i2pd starts → monerod routes through i2p automatically. No CLI, no separate install, no manual configuration required.
