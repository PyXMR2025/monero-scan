# daemon-rpc: incorrect doc fixes
author: hinto-janai
created_at: 2024-08-06T00:52:00Z
status: closed

### What
Fixes incorrect documentation for daemon RPC calls; requires review into `monerod`.

### Fixes
Table of fixes in this PR, originally tracked here: https://github.com/Cuprate/cuprate/issues/159.

| Route/endpoint | Link | Notes |
|----------------|------|-------|
| `get_miner_data` | [get_miner_data](https://www.getmonero.org/resources/developer-guides/daemon-rpc.html#get_miner_data), [rpc/core_rpc_server_commands_defs.h @ 1012](https://github.com/monero-project/monero/blob/cc73fe71162d564ffda8e549b79a350bca53c454/src/rpc/core_rpc_server_commands_defs.h#L1012) | `difficulty` field is noted as `unsigned int`, although it is actually a JSON string containing an unsigned int in hex form. Simple proof for `get_block`:
```bash
curl \
    http://127.0.0.1:18081/json_rpc \
    -d '{"jsonrpc":"2.0","id":"0","method":"get_block","params":{"height":0}}' \
    -H 'Content-Type: application/json'
```
