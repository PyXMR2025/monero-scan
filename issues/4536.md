## Feature: Enable I2P connectivity for enhanced anonymity 🚀


This Pull Request delivers a fully integrated I2P networking service, "baking in" the I2P functionality into the Monero GUI architecture. This addresses the critical need for an alternative, highly private connection transport, similar to how Bisq integrates Tor.

---

### 💡 High-Level Changes & Architectural Overview

The I2P feature is implemented as a modular service layer, ensuring a clean separation of concerns and maintainability.

* **New Service Layer:** Introduced dedicated C++ classes (`I2PManager.cpp/h`) to manage the I2P router, configuration, and status. This acts as the primary interface for the core Monero logic.
* **GUI Bridge:** Created the necessary $\text{C++}$ abstraction (`I2PNodeManager.cpp/h`) to expose I2P-specific properties and methods to the QML front-end.
* **User Interface:** Added a new **Dedicated Settings Page** (`SettingsI2P.qml`) for granular control over I2P connection mode and advanced configuration.
* **Seamless Integration:** Integrated I2P as a core option within the **Daemon Connection Wizard** (`WizardDaemonSettings.qml`), ensuring the enhanced privacy option is available from the first launch.
* **Build System:** Updated CMake configuration to support and link the necessary I2P libraries across all target platforms.

---

### ✅ Verification and Testing

This feature has been tested rigorously for stability and cross-platform compatibility.

* **Platforms Tested:** Built and manually tested successfully on **macOS**, and verification was performed across **Windows** and **Linux** CI/CD runners.
* **Functional Verification:** Confirmed that the wallet daemon successfully connects via the configured I2P tunnel and handles routing correctly.
* **Configuration Logic:** Verified that the I2P settings persist correctly across application restarts and wallet switches.
