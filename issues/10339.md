# Cannot Create Wallet - Device not found: Trezor
author: CepheidVariable
created_at: 2026-02-25T14:32:20Z
status: closed

I'm using the latest release from the monero-cli project: [release-v0.18](https://github.com/monero-project/monero/releases/tag/v0.18.4.5)

Installation steps:
- `sudo pacman -Syu --needed base-devel cmake boost openssl zeromq unbound libsodium libunwind xz readline expat python3 ccache doxygen graphviz qt5-tools hidapi libusb protobuf systemd`
- `git clone --recursive https://github.com/monero-project/monero`
- `cd monero && git checkout release-v0.18`
- `git submodule init && git submodule update`
- `make`

There were no error messages during compilation, that I saw.
The Trezor device works and is able to connect to the Trezor Suite using their supplied appimage.

When running monerod, and fully synced with the blockchain, I attempt to create a monero wallet:
`monero-wallet-cli --hw-device Trezor --generate-from-device <wallet_name>`

It prompts to enter a password for the new wallet. Once I enter and confirm, it fails with the following error:
"Error: failed to generate new wallet: device not found: Trezor"

Running lsusb shows the device:
Bus 003 Device 006: ID 1209:53c1 Generic SatoshiLabs TREZOR

Notes:
I've tried this process with and without the Trezor Suite running, and/or the device unlocked. Also, there seems to have been a "bridge" that was required in the past, but I confirmed with Trezor support that this is deprecated, and not longer required.
