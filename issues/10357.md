# Cleanup some of the fragmented levin handling
author: vtnerd
created_at: 2026-03-11T17:16:34Z
status: closed

The fragmented message handling in levin currently allows padding bytes to be sent to the `epee` storage parser. That parser ignores everything after the `epee` message. This upgrades the message handling so that the `epee` parser never sees the padding bytes in the fragmented message.

There are tons of unit tests related to this, including tx notify code. This basically proves that this can be added without disrupting existing p2p communications.
