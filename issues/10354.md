# 24-Hour Monero Node Monitoring Report (Tor node)

##  Overview
I ran a Monero node (v0.18.4.2) configured with Tor for 24 hours and collected detailed statistics about network behavior, connection patterns, and Tor reliability. All data was collected with privacy in mind (logs in RAM, anonymized before storage).

## Node Configuration
- **Monero version**: 0.18.4.2
- **Network**: Mainnet
- **Connection**: Tor (onion service)
- **Security**: 
  - Logs in RAM (`/dev/shm/`)
  - UFW firewall (RPC restricted to LAN)
  - `--restricted-rpc` enabled
  - `--prune-blockchain` for resource efficiency
- **Limits**: `--out-peers 8 --in-peers 4 --limit-rate 1000`

## Dataset
- **Duration**: 25 consecutive hours
- **Snapshots**: 25 (one per hour)
- **Collection method**: Automated script parsing node logs

## Key Findings

### Transaction Volume
- **Observation**: The network maintains remarkably stable transaction flow
- **No significant peaks or valleys** detected in this 24h window

### Tor Connection Failures
    Average: 11

    Maximum: 18

    Minimum: 0

    Total: 264 failures across 24h

- **Observation**: Failure rates vary but stay within expected range
- **Interesting**: One hour recorded **0 failures** - possible maintenance window?

### Peer Connectivity
    Average: 42

    Maximum: 48

    Minimum: 27

- **Observation**: Node maintains healthy connectivity even at minimum
- **Range**: 27-48 peers suggests good network diversity

## Patterns Detected

1. **Network Stability**: Transaction volume is remarkably consistent (1000/h)
2. **Tor Reliability**: ~11 failures/hour seems normal for Tor network
3. **Peer Distribution**: Even at minimum, node stays well-connected
4. **No Correlation Found**: Peers and failures don't appear directly correlated

## Methodology
- Logs stored in RAM only (`/dev/shm/monero/monero.log`)
- Hourly snapshots using custom bash script
- Python scripts for analysis and visualization
- All IPs anonymized (converted to /24 subnets)
- All transaction IDs hashed

##  Questions for the Community

1. Do these numbers align with what other node operators observe?
2. Is the constant 1000 TX/hour expected behavior?
3. What could explain the hour with 0 Tor failures?
4. Are there known Tor network maintenance windows?
