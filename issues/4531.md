Before implementing the i2p features, I performed a significant overhaul of the codebase. This included updating the **C++**, **Qt**, and **QML** to current, modern standards to improve performance and maintainability.

### Summary of i2p Changes:

* **C++ Logic:** The core wallet networking classes have been modified to route *all* wallet-to-daemon communication through a SOCKS5 proxy when i2p is enabled.

* **QML UI (Settings):**

  * Added a new top-level **"i2p" tab** in Settings.

  * Added the **"Enable i2p" toggle** to the "Interface" tab.

  * Added the **"Configure i2p" button** to the "Wallet" tab, which navigates to the new i2p page.

* **QML UI (i2p Page):**

  * The new "i2p" page contains inputs for `i2p Router Address` and `i2p Router Port`.

  * A "Test Connection" button is included to verify the i2p router is reachable.

* **Configuration:** All i2p settings (toggle state, address, port) are saved to and loaded from the wallet's configuration file.
