# daemon-rpc: doc undocumented fields
author: hinto-janai
created_at: 2024-08-08T21:08:37Z
status: closed

## What
Adds documentation for undocumented request/response fields within daemon RPC calls; requires review into `monerod`.

Some of these are lacking descriptions as I'm not sure what the behavior is for some of the fields. I will know after https://github.com/Cuprate/cuprate/issues/244 and may add more docs then.

### Other
The [`get_output_distribution`](https://www.getmonero.org/resources/developer-guides/daemon-rpc.html#get_output_distribution) method has 2 undocumented fields, [`binary` and `compress`](https://github.com/monero-project/monero/blob/cc73fe71162d564ffda8e549b79a350bca53c454/src/rpc/core_rpc_server_commands_defs.h#L2445-L2465). The current documentation is also incorrect as the output is not an array of `distribution` JSON objects, but rather, either raw binary or a JSON byte array. This may change with https://github.com/monero-project/monero/issues/9422 so it is not changed in this PR.
