# How do centralized exchanges find transactions for Monero subaddresses, and why is it complicated?
author: firstsoloplayer
created_at: 2025-09-22T10:47:30Z
status: closed

I’m working on a centralized exchange for Monero and ran into some issues with handling subaddresses, I’ve heard there are limitations when it comes to scanning large numbers of subaddresses. The exchange needs to generate a unique subaddress for each user,  suppose there are 100k users but scanning that many can take a long time or even cause missed transactions.

Another challenge is RPC performance calls like ``get_balance`` have to scan across all subaddresses, which becomes slow at scale.

Some people suggested setting up a new wallet every 1k subaddresses (using Docker), while others recommended using multiple accounts within the same wallet file for example, account_index = 0 with 1k subaddresses, then account_index = 1 with another 1k.

I’ve also heard suggestions to use payment IDs instead of subaddresses. I’d like to hear from people with real experience running Monero CEXs or similar custodial wallets on how they handle this.
