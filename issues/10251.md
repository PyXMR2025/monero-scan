# [Bug] Missing #include <cstdint>inhttp_base.h causes build failure with GCC 13+
author: alextodd1
created_at: 2025-12-10T21:36:33Z
status: closed

**Description**:

**Summary**
Cross-compiling Monero for Windows using the **depends** system fails due to a missing **#include <cstdint>** in **contrib/epee/include/net/http_base.h**. This causes **uint64_t** to be undefined when compiling **net_parse_helpers.cpp**.

**Environment**:

- Host OS: Ubuntu 24.04 (WSL2)
- Target: x86_64-w64-mingw32 (Windows 64-bit)
- GCC version: 13.0.0 (MinGW cross-compiler)
- Monero release: v0.18.4.4 

**Build command**:
```
make depends target=x86_64-w64-mingw32

### Error output

/home/alext/monero/contrib/epee/include/net/http_base.h:126:7: error: 'uint64_t' does not name a type
  126 |       uint64_t port;
      |       ^~~~~~~~
/home/alext/monero/contrib/epee/include/net/http_base.h:37:1: note: 'uint64_t' is defined in header '<cstdint>'; did you forget to '#include <cstdint>'?
   36 | #include <list>
  +++ |+#include <cstdint>
   37 |

/home/alext/monero/contrib/epee/src/net_parse_helpers.cpp: In function 'bool epee::net_utils::parse_url_ipv6(std::string, http::url_content&)':
/home/alext/monero/contrib/epee/src/net_parse_helpers.cpp:140:13: error: 'struct epee::net_utils::http::url_content' has no member named 'port'
  140 |     content.port = 0;
      |             ^~~~
```

**Cause**:
GCC 13+ is stricter about implicit header includes. The **uint64_t** type in **http_base.h** line 126 requires **<cstdint>** to be explicitly included, but the header only includes **<string>** and **<list>**.

**Workaround**:
Adding the missing include resolves the issue:
`sed -i '37a #include <cstdint>' contrib/epee/include/net/http_base.h`

**Suggested fix**:
Add #include <cstdint> to contrib/epee/include/net/http_base.h after line 36:
```
#include <string>
 #include <list>
+#include <cstdint>
 
 namespace epee
 {
```

After using quickfix the following compile successfully worked
```
make depends target=x86_64-w64-mingw32 \
    CMAKE_EXTRA_FLAGS="-DUSE_DEVICE_TREZOR=OFF -DBUILD_TESTS=OFF -DBUILD_DEBUG_UTILITIES=OFF" \
    CFLAGS="-O3 -march=znver3" \
    CXXFLAGS="-O3 -march=znver3"
```
