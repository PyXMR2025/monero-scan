# Allow requesting blocks without their transactions in `get_blocks.bin`
author: kayabaNerve
created_at: 2025-09-28T12:37:06Z
status: open

`get_blocks.bin` currently allows fetching blocks alone, the pool information alone, or both. When blocks are requested, the `block_complete_entry` is returned, including all transactions. It'd be _appreciated_ if I could request blocks _without_ their transactions, without having to use the JSON-RPC to request each block individually. Currently, I (within `monero-oxide`'s RPC code) do use the JSON-RPC to fetch blocks, hence why this would be appreciated (as it'd allow using a single request (as would #10118) _and_ reduce the size of the response by ~2x).

I'd be fine with either a new method or a new enum variant within `get_blocks.bin` for this.
