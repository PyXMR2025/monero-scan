# Daemon connection problems causes crash on wallet load
author: jeffro256
created_at: 2025-05-06T20:36:57Z
status: open

After Monero GUI opens a wallet (immediately after a successfully password prompt), it requests the unlocked balance from `wallet2`, which triggers an RPC call, which on failure, throws an exception which is uncaught by Monero GUI. This uncaught exception causes the entire program to crash. It isn't fixable without modifying the wallet cache file, since to change daemon settings, you need to open the wallet successfully ;).

When `tools::NodeRPCProxy::get_height` returns an error string, an exception of type `tools::error::wallet_internal_error` is thrown in `tools::wallet2::use_fork_rules`. This balance fetching thread in the GUI should anticipate some errors occurring in the called functions, and perform some graceful action on failure, i.e. setting the unlocked balance to some well-defined stand-in for "unknown".
