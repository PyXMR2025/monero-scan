# Starting monero-wallet-rpc with offline --daemon-address interferes with wallet functionality, even when overriden
author: woodser
created_at: 2025-12-02T20:44:24Z
status: open

The following issue is observed:

- Start monero-wallet-rpc with `--daemon-address` set to an offline address, e.g. "offline_server_uri" (this prevents monero-wallet-rpc from syncing automatically when a new wallet is initialized).
- Initialize a wallet (via RPC).
- Manually set an online daemon connection via the `set_daemon` RPC call, e.g. `localhost:28081`.
- The resulting wallet fails to function at least as a multisig wallet. After making the wallet multisig and synchronizing with participants, creating a transaction yields the error: "No transaction created". This error does not happen if monero-wallet-rpc is initialized with an online --daemon-address.

This issue requests that calling `set_daemon` fully overrides the startup daemon connection, so the resulting wallet functions correctly at least as a multisig wallet.
