# How to run monerod with valgrind
author: j-berman
created_at: 2025-11-03T19:00:54Z
status: closed

valgrind is a useful tool for debugging memory issues in `monerod`. It's referenced in the README as a debugging tool.

There are some tricky things to handle to get `monerod` running with valgrind. People have expressed these challenges in the past in other issues. Here is a list of steps I needed to do to get it "working" (tested on ubuntu and fedora 64-bit systems). Of note, it skips the PoW check when syncing. Hopefully this will be a useful point of reference for people who want to use valgrind with monerod.

### Instructions

1. Clone the valgrind git repo: `git clone https://sourceware.org/git/valgrind.git`

This is necessary to allow LMDB to open the database (see https://github.com/BVLC/caffe/issues/2404). It bumps the allowed max memory Valgrind can use to 1TB.

3. Install valgrind following these instructions: https://valgrind.org/downloads/repository.html.

4.  DANGER: this step will remove checking PoW on blocks when syncing.

There seems to be an issue with valgrind + RandomX compatibility: https://github.com/tevador/RandomX/issues/191. This side-steps the issue by dangerously allowing nodes to sync without checking PoW.

5. Build monerod and run as `valgrind /path/to/monerod`.

OPTIONAL: using `massif`

`valgrind --tool=massif --stacks=yes` will track heap and stack allocations over time, though warning, monerod will run pretty slow (massif on its own slows valgrind down a fair amount, and `--stacks=yes` makes it even slower). Once the program terminates, it will output a file `massif.out.*`. You can analyze the file with the program `massif-visualizer`, it'll open a nice intuitive GUI.
